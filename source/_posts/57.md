- 哨兵配置文件参考：

 	哨兵配置文件: etc/redis/sentinel.conf
	```			
	sentinel monitor test1 10.10.10.152 6386 1
	sentinel down-after-milliseconds test1 5000
	sentinel failover-timeout test1 900000
	sentinel can-failover test1 yes
	sentinel parallel-syncs test1 2
	
	sentinel config-epoch test1 45
	sentinel leader-epoch test1 45
	sentinel known-slave test1 10.10.10.155 6387
	
	port 26379
	dir "/usr/local/etc"
	sentinel current-epoch 45
	```

- 备注： 
	```
	redis哨兵10.10.10.155， 哨兵端口（26379）
	test1 10.10.10.152 6386  为其中一个监控的redis 主服务端口 
	```

- 前提：
 
	```
	哨兵：10.10.10.155
	redis端口 10.10.10.155 26379 实例启动 为哨兵服务
	命令：redis-server /usr/local/etc/sentinel.conf --sentinel

    本机：10.10.10.152
    redis端口 10.10.10.152 6386 实例启动 为主master服务，</br>
	命令：sudo redis-server /etc/redis/redis_master_6386.conf
	
    其他机：10.10.10.155
	redis端口 10.10.10.155 6387 实例启动 为从slave服务
	命令：sudo redis-server /etc/redis/redis_slave_6387.conf
	```

- 操作：

	```
    在10.10.10.155机器上命令：
    1. redis-cli -h  10.10.10.155 -p 6387 进入redis
    2. slaveof 10.10.10.152 6386  这样本机（10.10.10.155）作为主（10.10.10.152 端口 6386）的从数据库了
	```

- 测试要点：

	```
	1. 修改 27行 script_redis_monitor.php 文件 修改哨兵$_sentinels监控的IP和端口，修改为10.10.10.155 26379。
	2. 修改 141行 script_redis_monitor.php 文件 修改$address  为自己的邮件地址。
	3. 停掉哨兵服务 redis-server /usr/local/etc/sentinel.conf --sentinel 这个进程停掉，运行 php script_redis_monitor.php 看邮件报警。<span style="color:red">记得重新启动</span>。
	4. 停掉master服务sudo redis-server  /etc/redis/redis_master_6386.conf 这个进程停掉， <span style="color:red">3秒内</span> 运行 php script_redis_monitor.php 看邮件报警。<span style="color:red">记得重新启动</span>。
	5. 停掉master服务sudo redis-server  /etc/redis/redis_master_6386.conf 这个进程停掉， <span style="color:red">同时</span> 停掉slave服务sudo redis-server  /etc/redis/redis_slave_6387.conf 这个进程停掉，运行php script_redis_monitor.php 看邮件报警。
	```
